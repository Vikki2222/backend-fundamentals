# backend-fundamentals

backend-fundamentals/
│
├── scratch/
│   ├── app.js→ Entry point that initializes Express, sets middleware, registers routes, and starts the server.
│   ├── package.json→ Defines project metadata, dependencies, and runnable scripts.
│   ├── package-lock.json→ Locks exact dependency versions for consistent installs.
│   ├── .env.env → Stores environment variables and sensitive configuration values.
│   ├── .gitignore→ Specifies files and folders Git should not track.
│   │
│   ├── config/→ Stores configuration logic like database connections and environment setup.
│   ├── controllers/ → Contains business logic for handling requests and sending responses.
│   ├── middlewares/→ Holds reusable functions that run between request and response.
│   ├── models/ → Defines database schemas and handles data interaction.
│   ├── routes/ → Defines API endpoints and maps them to controllers.
│   ├── utils/→ Contains helper and utility functions shared across the app.
│   ├── views/→ Stores server-side templates (EJS) for rendering dynamic HTML.
│   ├── public/ → Contains static assets like CSS, JavaScript, and images.
│   └── node_modules/→ Automatically generated folder containing installed dependencies.()will not be uploaded in github)
│
└── README.md→ Documents the project purpose, setup, and structure.
